# Daily Process Integration

How mirror-palace frameworks feed into daily practice. These templates turn abstract frameworks into concrete daily rhythms.

## Files

- **briefing-template.md** — Morning briefing that incorporates framework-aware prioritization
- **reflection-template.md** — Evening/weekly reflection using frameworks to extract patterns
- **check-in-protocol.md** — Quick energy/state check-in before starting work
- **framework-of-the-day.md** — Rotating framework spotlight to build familiarity over time

## Philosophy

Frameworks only work if they're embedded in routine. A framework you read once and never apply is worse than useless — it creates the illusion of understanding without the reality of behavior change.

These daily templates create low-friction touchpoints where frameworks get applied to real life. The goal isn't to spend 30 minutes on self-reflection every morning. It's to spend 2-3 minutes checking in, and letting the system do the heavy lifting of connecting today's reality to the right framework.

## Integration with Status System

Daily processes are the primary mechanism for keeping `status/` current:
- Morning briefing surfaces stale domains (last updated >30 days)
- Reflections generate updates to people records and decisions
- Check-ins capture energy data for health-fitness domain
