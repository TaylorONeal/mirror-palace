# System Anti-Patterns

Twelve design patterns that appear reasonable but consistently produce the opposite of their intended effect: showing the full backlog, equal-weight options, shame language, fixed rewards only, assuming motivation, ignoring energy, instant transitions, valueless notifications, the dashboard trap, feature bloat, data hoarding, and one-size-fits-all. Each with why it fails psychologically and what to do instead. Applies to software, personal routines, team processes, and organizational structures.

## Quick Reference
- **Best for**: Auditing and improving systems that interact with human psychology
- **Complexity**: Medium (12 patterns, each with diagnosis and counter-design)
- **Time to apply**: 20-30 minutes for a full system audit
- **Pairs with**: Behavior Equation (friction analysis), Variable Reward Schedules (alternative to fixed rewards), Developmental Stages (user-appropriate design)
