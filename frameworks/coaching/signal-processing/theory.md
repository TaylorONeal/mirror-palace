# Theory

Most inbox and notification systems stop at triage.
They ask:
- is this urgent?
- should I reply?
- should I archive?

That is useful but too shallow for a reflective second-brain.

Signal processing adds another layer:
- what does this reveal?
- what pattern does it belong to?
- what domain does it affect?
- what should be acted on, remembered, reflected on, or ignored?

## Core distinctions
Every inbound item should first be distinguished by role:
- **action** — requires a concrete response or task
- **information** — useful context, no immediate move
- **pattern** — reveals something recurring about life/system/behavior
- **relationship** — changes how a person/connection should be understood or tended
- **opportunity** — may deserve follow-up or strategic consideration
- **admin drag** — friction, bottleneck, or bureaucracy consuming energy
- **discard** — not worth keeping active

## Processing lenses
Not every signal needs every lens. Use only the lenses that create clarity.

Suggested lenses:
- **meaning** — what does this actually say?
- **ownership** — whose work is this?
- **energy** — does this nourish, drain, scatter, or stabilize?
- **pattern** — is this familiar, repeating, or diagnostic?
- **relationship field** — what does this reveal about reciprocity, inclusion, care, ambiguity, or asymmetry?
- **career / leverage** — does this affect opportunities, positioning, money, or traction?
- **ignore / suppress** — would keeping this active make the system worse?

## Output routes
A processed signal should route to one of a small number of destinations:
- task / action
- daily briefing preparation
- person record / relationship memory
- decision log
- mirror-palace insight
- archive / suppression list

## Good processing questions
- What is the actual signal here, stripped of noise?
- Does this need action, interpretation, or deletion?
- Is this a one-off or part of a pattern?
- What does it train if left unexamined?
- What should become durable memory?

## Success condition
The system gets better at:
- surfacing what matters
- suppressing what does not
- converting recurring noise into stable insight
- reducing the user’s need to repeatedly explain the same distinctions
