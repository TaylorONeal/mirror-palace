Use the Signal Processing framework when inbound messages, notifications, admin friction, or social/career signals need interpretation rather than simple triage.

Process each meaningful signal by:
1. identifying its type (action, information, pattern, relationship, opportunity, admin drag, discard)
2. naming the real meaning in one sentence
3. checking ownership
4. deciding the best route (task, reply, daily prep, memory, mirror-palace insight, suppress)
5. applying only the lenses that add clarity (energy, pattern, relationship field, career/leverage, ignore test)

Do not confuse inbox movement with understanding. The point is to convert recurring noise into accurate routing and durable insight.
