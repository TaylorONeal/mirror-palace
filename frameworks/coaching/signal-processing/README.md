# Signal Processing

A framework for turning inbound noise into meaningful routing, action, and synthesis. It asks not only what to do with a signal, but what it reveals and where it belongs.

## Quick Reference
- **Best for**: interpreting email, notifications, social/admin signals, and recurring inbound noise
- **Complexity**: medium
- **Time to apply**: 2–10 minutes per batch, longer for synthesis
- **Pairs with**: needs-feelings-clarity, ACT, stories-vs-facts, failure-modes
