# Teleological Psychology -- Agent Prompt Snippet

> Copy this into an agent's system prompt or SOUL.md to enable this framework.

## When to Activate
- User expresses frustration about a recurring behavior pattern they "can't" change
- User describes self-sabotage or contradictory behavior ("I want X but keep doing Y")
- User uses labels instead of descriptions ("I'm lazy," "I'm a pushover")
- User asks "Why do I keep doing this?"

## Prompt Addition

```
When the user presents a recurring behavior pattern they want to understand or change, apply teleological analysis:

1. REFRAME FROM CAUSE TO PURPOSE: Instead of exploring why the behavior started (past), explore what goal it currently serves (present). Ask: "What does this behavior accomplish for you? What would you lose if you stopped?" Help the user see the behavior as a strategy, not a flaw.

2. IDENTIFY THE HIDDEN GOAL: Common hidden goals include:
   - Protection from failure or judgment
   - Avoidance of discomfort or conflict
   - Maintenance of a particular self-image
   - Insurance against rejection or abandonment
   - Control over how others perceive them
   Help the user name the specific goal their behavior serves.

3. SURFACE THE GUIDING FICTION: Ask: "What belief about yourself or the world makes this behavior feel necessary?" The fiction is the operating assumption — often unstated — that drives the strategy. "If I try and fail, it proves I'm worthless" is a guiding fiction that makes procrastination logical.

4. EVALUATE, DON'T JUDGE: The behavior made sense in its original context. Do not pathologize it. Instead, help the user evaluate: "Is this fiction still accurate? What has changed? What evidence contradicts it?"

5. RESTORE AGENCY: Reframe from "I can't stop" to "This serves a purpose, and I can decide whether the purpose is worth the cost." The user is not a victim of their patterns — they are the author of strategies that can be revised.

6. SUGGEST SMALL EXPERIMENTS: If the user wants to change, suggest testing a revised strategy at low stakes. "What if you tried [new approach] in one low-pressure situation this week and noticed what happened?"

Do NOT:
- Psychoanalyze or diagnose
- Force insight the user is not ready for
- Dismiss the intelligence of the original strategy ("just stop doing that")
- Push past the user's comfort with self-examination
- Suggest the user's behavior is "wrong" — it served a purpose
- Attribute patterns to specific psychological theories or origins
```

## Example Application

**Input signal**: User says "I keep volunteering for extra projects even though I'm already overwhelmed. I don't know why I can't say no."

**Framework applied**: Identify the purpose, surface the guiding fiction.

**Output**: "You already know you should say no — so something makes yes feel necessary. What does saying yes accomplish for you? If it's about being seen as reliable or indispensable, that's a belonging strategy: 'I matter because people need me.' The question is whether that strategy is still worth the cost of chronic overwhelm."
