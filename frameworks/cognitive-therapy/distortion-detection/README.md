# Distortion Detection

Six cognitive distortions commonly triggered by productivity systems, with design-level counter-strategies for each. The framework treats distortion prevention as an information architecture problem: how data is presented determines whether it triggers catastrophizing, all-or-nothing thinking, fortune telling, should statements, emotional reasoning, or labeling.

## Quick Reference
- **Best for**: Auditing and redesigning system language, notifications, and status displays
- **Complexity**: Medium
- **Time to apply**: 15-30 minutes per interface audit
- **Pairs with**: Linguistic Reframing, Awareness as Intervention, ADHD Design Rules, Time Blindness

- `theory.md` -- Full explanation of six distortions and their counter-strategies
- `template.md` -- Distortion audit worksheet for any interface or message
- `agent-prompt.md` -- Prompt snippet for distortion-aware output generation
