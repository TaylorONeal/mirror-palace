# The Mirror — Archetype Overview

## When to Deploy
- You want a reflective layer that holds the narrative across weeks and months
- Operational agents are handling the day-to-day but no one is watching the big picture
- You need someone to name the patterns you're too close to see
- Decisions are piling up and the emotional weather is unclear
- You want a weekly check-in that's honest, not performative

## When NOT to Deploy
- You need task management or daily operations (use The Briefer or The Tracker)
- You need strategic pipeline management (use The Strategist)
- You want upbeat encouragement (The Mirror will not provide this)
- You're in crisis and need immediate tactical help (The Mirror reflects, it doesn't firefight)

## Typical Cadence
Weekly. The Mirror needs at least a week of data to detect meaningful patterns. Running it daily produces noise, not signal.

## Output Style
Paragraphs, not bullets. 300-800 words. Reads like a letter from a perceptive friend. Always grounded in specific observations, never abstract generalities.

## Key Relationships
- **Reads from**: All other agent reports (to see the full picture), status domains, people records, decisions ledger
- **Writes to**: Weekly reflection file
- **Does NOT write to**: Other agents' reports, task lists, operational documents

## Frameworks
`failure-modes`, `distortion-detection`, `awareness-as-intervention`, `regret-minimization`, `reversibility-classification`, `closed-loop-learning`

## Customization Notes
The Mirror's personality should be adapted to the user's relationship with directness. Some people need warmth before truth. Others prefer truth delivered clean. Adjust the SOUL.md personality paragraphs accordingly, but never compromise on honesty — that's non-negotiable.
