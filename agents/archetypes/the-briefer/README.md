# The Briefer — Archetype Overview

## When to Deploy
- You have multiple agents or data sources and need one place to see what matters
- You want a daily synthesis that respects your time
- Information overload is a real problem and you need aggressive filtering
- You want alignment checking: does how you spend your time match what you say matters?

## When NOT to Deploy
- You only have one agent (no cross-agent synthesis needed)
- You want depth over breadth (use The Mirror for deep reflection)
- You need strategic planning (use The Strategist)
- You prefer to read all reports yourself

## Typical Cadence
Daily, once in the morning. Can add an evening wrap-up for high-intensity periods. The Briefer should not run more than twice a day — that would defeat the purpose.

## Output Style
Structured bullets. 3-5 priority items. Under 300 words. Headers for each priority. One sentence per item unless escalation is needed.

## Key Relationships
- **Reads from**: ALL other agent reports, status indexes, decisions index
- **Writes to**: Daily briefing file (e.g., `shared/daily-briefing.md`)
- **Does NOT write to**: Individual agent files, status records (other agents own those)

## Frameworks
`north-star-test`, `information-compression`, `executive-function-model`

## Customization Notes
The number of priority items can be adjusted. Some people want "top 3", others want "top 5." The voice can range from military-crisp to conversational-efficient. What should not change: the ruthless filtering. If The Briefer starts including "nice to know" information, it has failed at its core job.
