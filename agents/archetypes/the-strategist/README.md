# The Strategist — Archetype Overview

## When to Deploy
- You have multiple goals or projects in flight and need pipeline management
- Items are stalling without anyone noticing
- You want leverage analysis — where to invest effort for maximum return
- Priority drift is a recurring problem (saying one thing, doing another over weeks)
- You need someone to tell you what to stop doing, not just what to start

## When NOT to Deploy
- You have one clear focus and don't need pipeline management
- You want emotional or relational insight (use The Mirror)
- You need daily tactical prioritization (use The Briefer)
- You prefer to manage strategy yourself without agent input

## Typical Cadence
Daily to weekly, depending on how many active pipelines exist. Daily during high-activity periods (job search, project launches). Weekly during stable execution phases.

## Output Style
Pipeline stages with status. Stall alerts. Leverage comparisons. 200-500 words. Structured but with enough narrative to explain strategic reasoning.

## Key Relationships
- **Reads from**: Status domains (especially career-work), decisions index, project state, tracker reports
- **Writes to**: Strategy report file (e.g., `shared/strategy-report.md`)
- **Feeds into**: The Briefer reads strategy reports for daily synthesis. The Mirror reads them for long-arc pattern analysis.

## Frameworks
`jobs-to-be-done`, `regret-minimization`, `north-star-test`

## Customization Notes
The Strategist works best when goals have defined stages and timelines. Before deploying, establish the pipeline stages you use (the default is exploring → active → committed → completed) and define what "stalled" means for your context (default: no movement in 14+ days). Leverage scoring can be subjective — calibrate it to your situation.
